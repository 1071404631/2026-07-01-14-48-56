# 2026-07-03 工作日志

## Vercel 部署上线后的 Bug 修复

### 问题1：左侧已提交职位列表不显示
- 原因：`api/submissions.js` 中按姓名过滤时用 `r.submitterName`，但实际数据结构是 `r.submitter.name`
- 修复：改为 `r.submitter && r.submitter.name === name`

### 问题2：Logo 图片不显示
- 原因：`vercel.json` 使用旧版 `builds` + `routes` 配置，catch-all 路由干扰了静态文件服务
- 修复：改为新版 `rewrites` 配置，只重写 API 路由，静态文件（HTML/PNG）由 Vercel 自动服务
```json
{
  "rewrites": [
    { "source": "/api/submit", "destination": "/api/submit.js" },
    { "source": "/api/submissions", "destination": "/api/submissions.js" }
  ]
}
```

### 问题3：后台删除记录无反应
- 原因1：`admin.html` 发送 `DELETE /api/submissions/{id}` 路径参数，但 Vercel 函数不认这个路径
- 原因2：`api/submissions.js` 只处理了 GET，没有 DELETE 逻辑
- 修复：
  - `admin.html` 改为 `DELETE /api/submissions?id=xxx` 查询参数
  - `api/submissions.js` 增加 DELETE handler，从 JSONBin 读取全部记录，过滤掉指定 ID 后写回

### 问题4：Vercel 500 报错「Serverless Function has crashed」
- 原因：`api/*.js` 使用了 ESM 语法 `export default`，但项目 `package.json` 没有 `"type": "module"`，Vercel Node 函数默认按 CommonJS 加载，导致模块解析崩溃
- 修复：把 `export default` 改为 `module.exports =`
- 另外保留 `vercel.json` 的 `functions` 显式声明和 JSONBin 响应状态检查

### 问题5：删除仍失败（前端报错「Unexpected token 'I'... is not valid JSON」，后端管理页无反应）
- 原因：
  - `index.html` 前端使用 `DELETE /api/submissions/${id}` 路径参数方式，但 `api/submissions.js` 只识别 `?id=xxx` 查询参数，导致路径参数请求走到 Vercel 404 HTML 页面，JSON 解析报错
  - 管理后台 `admin.html` 的 `?id=xxx` 请求已走到函数，但此前函数对路径参数的兼容不足，且编辑功能使用的 `PUT /api/submissions/${id}` 也未被支持
- 修复：
  - `api/submissions.js` 改为同时兼容路径参数 `/api/submissions/xxx` 和查询参数 `?id=xxx` 两种传参方式
  - 新增 `PUT` handler，支持更新已有记录（保留原 id 和 createdAt）
  - 增加 `Access-Control-Allow-Methods: GET, DELETE, PUT, OPTIONS`
  - `index.html` 删除和编辑统一改为查询参数方式，并增强非 JSON 响应的错误提示
  - 请求体读取兼容 Vercel 自动解析和原始流两种模式
- 文件改动：`api/submissions.js`、`index.html`

### 问题6：推送后线上仍报 500 / 删除仍失败
- 现象：curl 调用 `/api/submissions`、`/api/submit` 均返回 `FUNCTION_INVOCATION_FAILED`；前端删除仍报 JSON 解析错误
- 排查：GitHub 上代码已更新，但线上函数似乎仍在崩溃，怀疑 Vercel 运行时对 `fetch` 的支持或部署状态不稳定
- 修复：
  - 将 `api/submit.js` 和 `api/submissions.js` 中的 `fetch` 全部改为 Node 原生 `https` 模块，彻底消除运行时差异
  - 响应发送改为原生 `res.statusCode` + `res.end(JSON.stringify(...))`，避免 Vercel 响应辅助方法潜在问题
  - 请求体读取兼容 Vercel 自动解析和原始流，并排除 Buffer/Uint8Array 误判
  - `submissions.js` 仍同时兼容路径参数和查询参数
- 文件改动：`api/submit.js`、`api/submissions.js`
